Welcome

              

                Welcome to the SimNexus / ETS2 Save Edit Tool. This wiki explains the main areas, shortcuts and features directly inside the app.
              

            


            

              
Shortcuts

              

                

                  F1
                  Open the Truck section.
                

                

                  F2
                  Open the Trailer section.
                

                

                  F3
                  Open the Profile section.
                

                

                  F4
                  Open the Settings section.
                

                

                  ESC
                  Close open modals or windows when possible.
                

              

            


            

              
Save Edit Tool

              
    
                
  • Edit truck values such as wear, fuel, mileage and identity fields.
    • 
                
  • Edit trailer values such as condition, cargo mass and trailer identity data.
    • 
                
  • Edit profile values such as money, XP, level and related progress data.
    • 
                
  • Review values before writing changes to the active save.
    • 
                
  • When backup and restore workflows are available, use them before applying risky changes.
    • 
              

            


            

              
Career Mode / VTC

              
    
                
  • Dispatcher workflows help coordinate job generation, assignment and routing decisions.
    • 
                
  • Jobs, offers, active contracts and history provide operational visibility for current and past work.
    • 
                
  • The company profile area covers company identity, visibility and organization settings.
    • 
                
  • Members and roles define access, responsibilities and company participation.
    • 
                
  • Fleet panels summarize trucks, assignments, readiness and maintenance state.
    • 
                
  • Balance, statistics and achievements help track long-term company progress.
    • 
              

            


            

              
Save Health Monitor

              
    
                
  • Save health can report states such as Clean, Risky or Broken.
    • 
                
  • Missing mods can lead to incomplete loads or unstable save behavior.
    • 
                
  • Broken references indicate objects or links that can no longer be resolved safely.
    • 
                
  • Inconsistent XP or level values can produce progression errors after editing.
    • 
                
  • Review hints and proposed fixes before applying any automated repair step.
    • 
              

            


            

              
Auto Backup & Restore

              
    
                
  • Automatic snapshots can be created before write operations.
    • 
                
  • Restore workflows allow returning to a previous save state when needed.
    • 
                
  • If a diff preview is available, inspect it before restoring or overwriting data.
    • 
                
  • Backups matter because save edits can affect progression, references and game stability.
    • 
              

            


            

              
Logging & Error Reports

              
    
                
  • User-facing error reports should explain what failed in a readable way.
    • 
                
  • Use Copy Report when you need to share diagnostic output quickly.
    • 
                
  • Use Export Logs when a longer support or debugging trail is required.
    • 
                
  • When both are available, user logs focus on workflows while developer logs include deeper technical traces.
    • 
              

            


            

              
Updates

              
    
                
  • Check for updates from inside the app when update functionality is available.
    • 
                
  • Download the update package before installation starts.
    • 
                
  • Install updates with the built-in flow instead of replacing files manually.
    • 
                
  • Relaunch the app after installation if prompted.
    • 
                
  • Use the progress display to monitor download and install status.
    • 
              

            


            

              
Accessibility / Bedienung

              
    
                
  • Most core actions should remain reachable with keyboard navigation.
    • 
                
  • Visible focus states make navigation and active controls easier to track.
    • 
                
  • Readable contrast helps when working for long sessions in dark UI themes.
    • 
                
  • The help button keeps this wiki reachable at any time inside the app.
    •
